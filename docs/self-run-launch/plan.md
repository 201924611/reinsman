# STEP 3 — 실행 계획서: "The Self-Run Launch" (C4 + 보조 C5)

_채택안: 홍보 캠페인 자체를 agent-core가 goal로 실행하고, 머신검증 가능한 trace/eval 증거 + 원커맨드 재현 + 에이전트 저자성(provenance)을 무기로 삼는다._

## 0. 핵심 서사 (모든 문안의 뼈대)
> "이 런치를 쓴 건 사람이 아니라 agent-core 자신이다. 믿지 말고 **검증**하라 — 실행 trace(도구체인·서브에이전트·토큰)와 eval 채점을 첨부했고, **한 줄 명령으로 당신도 똑같이 재현**할 수 있다."
- 안티슬롭 프레임: "AI가 썼다"를 숨기지 않고 **증거로 뒤집는다**. 회의론이 곧 데모 트리거.

## 1. 단계 (Phase)
- **P0 (준비, D0)**: 레포 위생 — LICENSE(MIT) 추가, README 상단에 "PROVENANCE" 배지/문단, `PROVENANCE.md`, 재현 스크립트, 데모 GIF 캡처 지점 확보. (STEP4 자산으로 생성)
- **P1 (증거화, D0)**: agent-core로 실제 goal 1건 실행 → `traces/`·`evals/` 산출물을 `docs/evidence/`로 스냅샷(민감정보 스크럽). 이 goal이 "홍보물 작성" 자체여도 됨(메타 증거).
- **P2 (허브 공개, D1)**: GitHub README + PROVENANCE + Public Live Ledger(C5) 공개판을 repo에 커밋(사람 1클릭: push).
- **P3 (배포, D1~D3)**: 표준 채널을 '그릇'으로만 사용 — Show HN, r/selfhosted/r/LocalLLaMA, X 스레드, awesome-list PR. 단 **문안·훅·아티팩트는 전부 C4 서사**. 각 채널 최종 문안은 STEP4에서 완성.
- **P4 (측정·회수, D3~D14)**: 지표 수집(아래) → 상위 반응 채널에 후속(에이전트가 이슈/댓글 답변을 goal로 실행 = C6 미래 확장).

## 3. 채널 & 이유
- **Show HN**: 기술 청중이 '재현 가능한 증거'라는 훅에 가장 반응. 제목에 provenance 각도 명시.
- **r/selfhosted + r/LocalLLaMA**: self-hosted 자율 하네스 = 정확한 타깃. 로컬/구독 구동(추가비용0)이 셀프호스터에 강점.
- **X 스레드**: GIF + "이 스레드도 에이전트가 초안" 링크로 매체=메시지 일치.
- **awesome-list PR**: 장기 발견성/SEO(위생 요소).
- (모두 스팸 금지: 각 커뮤니티 1회 진정성 포스트, 규칙 준수, 자기홍보 표기.)

## 4. 성공지표 (측정 방법 STEP5에 상술)
- **1차(도달)**: GitHub 스타 증가(Δstars), traffic(Insights→Traffic: views/unique/clones), referrer별 유입.
- **2차(참여)**: 이슈/디스커션 유입 수, README의 재현 스크립트 실행 흔적(clones), Show HN 포인트/댓글, Reddit upvote/코멘트.
- **3차(품질/전환)**: '재현 성공' 보고 수, star→watch 전환, 실제 fork.
- **목표(정직한 베이스라인, 무명 개인)**: D14까지 스타 +25~100, HN 프론트 진입은 보너스(확률 낮음), 이슈/디스커션 3건+, 재현 성공 보고 1건+ 이면 서사 검증 성공.
- **novelty 성공 판정**: '재현했다/trace 봤다'는 반응이 1건이라도 나오면 = 남이 못 베끼는 각도가 작동한 증거.

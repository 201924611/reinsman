# STEP 2 — novelty 후보 + 게이트 (★1~3, 자기반박, 채택/탈락)

_규칙: 각 후보에 novelty ★1~3 + '왜 이제까지 없었나' 1줄 + 스켑틱 자기반박(정말 새로운가/누가 했나, 웹검증). 상위 1~2만 채택._

## 후보 목록

### C1. 표준 런치 세트 (Show HN + r/selfhosted + X 스레드) — ★1
- 왜 새롭지: 안 새롭다. research.md B-1~3 그대로.
- 스켑틱: 포화. 무명 개인은 도달 0. **탈락**(단, 채택안의 '배포 채널'로는 재활용 — 형식이 아니라 그릇).

### C2. awesome-list PR + 데모 GIF — ★1
- 위생 요소. 발견성/SEO엔 유효하나 그 자체론 화제성 0.
- 스켑틱: 표준 관행. **탈락(주력) → 보조 자산으로만 포함.**

### C3. "에이전트가 자기 changelog를 씀" — ★1 (함정)
- 겉보기 새로움: 제품이 자기 릴리즈노트를 자동 작성.
- 스켑틱 웹검증: **이미 흔함.** BunnyDesk·Taskade·GitHub Actions·releaseglow 등 커밋→AI 요약 changelog 툴이 이미 상용. AI 생성 콘텐츠는 오히려 '슬롭' 부정신호가 될 수 있음. → **탈락.** (교훈: 'AI가 썼다'는 novelty가 아니다. 이미 레드오션.)

### C4. ⭐ "The Self-Run Launch" — 홍보 캠페인 자체를 하나의 goal로 에이전트가 실행하고, **머신검증 가능한 실행 증거(trace/eval JSON + LEDGER)와 원커맨드 재현**을 첨부 — ★3
- 왜 이제까지 없었나: 기존 툴(C3)은 *커밋을 요약하는 좁은 봇*이다. agent-core는 *모호한 인간 goal("너를 홍보해라")을 서브에이전트·자가도구생성까지 동원해 end-to-end로 완수*하고, 그 **실행 자체가 제품의 증거**가 된다. 홍보물=제품의 실행 로그=재현 가능한 증거로 매체와 메시지를 일치시킴. changelog봇은 자율 goal 실행 trace를 낼 수 없다(애초에 goal 실행기가 아님).
- 스켑틱 자기반박:
  - "AI가 글 쓴 건 흔하잖아?" → 맞다. 그래서 novelty의 핵심은 'AI가 썼다'가 **아니라** **검증가능한 provenance + 재현성**이다. 청중이 신뢰하지 않아도 `trace/<task>.json`(토큰·도구체인·서브에이전트 그룹)과 `evals/<task>.json`(LLM judge 채점)로 **직접 검증**하고, 원커맨드 goal로 **직접 재현**한다. '주장'이 아니라 '증거'.
  - "build-in-public도 흔하잖아?" → 인간이 쓰는 build-in-public과 달리 **저자가 에이전트 본인**이고 각 문장이 trace로 소급 검증된다. 슬롭 리스크는 '이건 AI 슬롭이 아니라, 자율 실행의 증거물'이라는 프레임 전환으로 방어(회의론을 무기로 전환).
  - "누가 이미 했나?" 웹검증 결과 changelog 자동화는 있으나, **자율 하네스가 자기 런치를 goal로 돌리고 trace/재현을 증거로 첨부한 사례**는 검색상 관측 안 됨. 베끼려면 같은 자율+trace/eval 하네스가 있어야 하므로 방어적.
- **채택 (주력).**

### C5. ⭐ "Public Live Ledger" — 에이전트가 매 사이클 스스로 갱신하는 공개 build-in-public 레저(이 LEDGER.md의 공개판) — ★2
- 왜 새롭나: 인간 build-in-public이 아니라 **에이전트가 저자인 연속 작업 서사** + 체크포인트가 재개가능성(하네스 특성)을 그대로 홍보 소재로. "에이전트가 멈췄다 이어서 일하는 모습"을 공개.
- 스켑틱: 단독으론 약함(그냥 로그로 보일 수 있음). C4에 **종속 채널**로 붙이면 강함(C4의 증거를 지속 노출하는 상설 창구). → **조건부 채택: C4의 하위 채널로 편입.**

### C6. 에이전트가 자기 GitHub 이슈에 답변(dogfood 이슈 트레일) — ★2
- 새롭지만 실제 이슈 유입 + 외부 포스팅 필요(가드레일상 우리가 직접 못 함). → **탈락(현 단계 실행 불가) → 채택안 로드맵의 '3단계 확장'으로만 메모.**

## 최종 채택
- **주력: C4 "The Self-Run Launch"** (provenance + 재현성). ★3.
- **보조 채널: C5 "Public Live Ledger"** (C4 증거의 상설 노출 창구). ★2.
- 배포 그릇은 C1(Show HN/r/selfhosted/X)·C2(awesome PR/GIF)를 **형식으로만** 재사용하되, 그 안의 메시지·아티팩트는 C4로 채운다.

## 탈락 사유 요약
- C1·C2: 포화·위생요소 → 주력 불가, 채널/보조로만.
- C3: 웹검증상 이미 레드오션(changelog봇) + 슬롭 부정신호 → 탈락.
- C6: 외부 포스팅·실제 이슈 필요(가드레일 위반) → 미래 로드맵으로만.
